# Contributing to ToolBits

Thanks for your interest in contributing to ToolBits! We welcome improvements of all sizes: bug fixes, new tools, performance improvements, accessibility fixes, and documentation updates.

## Ways to Contribute

- Report bugs
- Suggest features
- Improve documentation
- Fix existing issues
- Build new utility tools

## Development Workflow

1. **Fork** the repository and clone your fork.
2. Create a branch:

   ```bash
   git checkout -b feat/your-change
   ```

3. Install dependencies and run locally:

   ```bash
   pnpm install
   pnpm dev
   ```

4. Run checks before opening your PR:

   ```bash
   pnpm lint
   ```

5. Commit with a clear message and open a pull request.

## Pull Request Guidelines

- Keep changes focused and scoped.
- Include a clear summary of what changed and why.
- Reference issues when applicable.
- Add screenshots for UI changes when useful.
- Ensure your branch is up to date with the base branch.

## Code Style

- Use TypeScript and existing project patterns.
- Keep components readable and modular.
- Prefer small, composable functions.

## Community

Please be respectful and constructive in discussions and reviews.

Thanks for helping make ToolBits better.
